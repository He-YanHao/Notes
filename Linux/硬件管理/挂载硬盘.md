# 挂载硬盘

## 自动挂载

通过修改`/etc/fstab`文件实现自动挂载。

```shell
hing@he-books ~> cat /etc/fstab
#Ubuntu
UUID=edc0b12d-811a-4f7c-8726-04d98b9da753  /                           xfs       defaults   0 1
UUID=9bc2a7d5-e20f-4a13-94d9-402b3e196d78  /home                       xfs       defaults   0 2
UUID=7010048c-b612-4834-b2f0-73afdf6ad7ca  none                        swap      sw         0 0

#Windows
UUID=CA9A27EB9A27D2AD                      /home/hing/Windows/System   ntfs3     defaults   0 2
UUID=60FA2097FA206B8A                      /home/hing/Windows/Data     ntfs3     defaults   0 2

#Arch
UUID=5f526c7e-7f43-494e-9c6f-69ae856537df  /home/hing/Arch/System      xfs       defaults   0 2
UUID=bfb0c79d-343b-4b82-87dd-41cdba13b487  /home/hing/Arch/Data        xfs       defaults   0 2
```

使用

```shell
sudo lsblk -f
```

可以查看硬盘的UUID，这是硬盘分区的唯一编号。

格式依次是：

```shell
UUID=???  挂载点  文件系统格式  挂载选项  备份  检查顺序
```

-   defaults：这是一个综合选项，包含了 `rw, suid, dev, exec, auto, nouser, async`。对于大多数情况，这样写就够了。

-   **`0`**: dump 备份工具是否备份此文件系统。0 表示不备份，1 表示备份。通常设为 0。

-   **`2`**: 系统启动时 `fsck` 磁盘检查的顺序。根目录 `/` 必须为 1，其他文件系统应为 2。0 表示不检查。
