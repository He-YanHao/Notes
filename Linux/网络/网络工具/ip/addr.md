# 一、整体结构说明

`ip addr` 是 `iproute2` 工具（Linux 标准网络工具）输出的网卡信息。

每一块网络接口都会显示：

```
编号: 接口名: 状态
    MAC地址
    IPv4
    IPv6
```

------

# 二、逐个解释你的输出

------

## ① lo — 回环接口（非常重要）

```
1: lo: <LOOPBACK,UP,LOWER_UP>
inet 127.0.0.1/8
inet6 ::1/128
```

### 它是什么？

回环接口（本机内部通信）。

用途：

```bash
ping 127.0.0.1
```

这是自己和自己通信。

特点：

- 不经过网卡
- 不经过路由器
- 不走WiFi

它是操作系统内部虚拟网络。

------

## ② eno1 — 有线网卡

```
2: eno1: <NO-CARRIER,BROADCAST,MULTICAST,UP> state DOWN
```

### 状态解释：

| 标志       | 意义         |
| ---------- | ------------ |
| NO-CARRIER | 没插网线     |
| UP         | 接口启用     |
| state DOWN | 没有物理连接 |

你的有线网卡当前：

👉 没插网线
👉 物理层没有信号

------

## ③ wlp4s0 — WiFi 网卡（当前主用）

```
3: wlp4s0: <UP,LOWER_UP>
inet 192.168.1.164/24
```

这是你当前真正联网的接口。

### 重要信息：

```
192.168.1.164/24
```

意思是：

- 你的局域网IP = 192.168.1.164
- 子网掩码 = 255.255.255.0

你现在所有网络流量都走它。

------

### IPv6 地址（可忽略）

```
2409:8a44:....
```

这是公网 IPv6 地址。

Ubuntu 默认启用 IPv6。

你现在学习阶段：

👉 先专注 IPv4 即可。

------

## ④ dae0@if4 — 虚拟接口

```
5: dae0@if4:
link-netns daens
```

这个是：

👉 网络命名空间（network namespace）接口

可能来源于：

- Docker
- 虚拟网络
- 容器环境
- VPN

它不是物理网卡。

------

# 三、你当前网络结构总结

```
你电脑
    ↓
wlp4s0 (WiFi)
    ↓
路由器
    ↓
互联网
```

有线网卡（eno1）没插。

# 四、你现在可以做的实验

## 1️⃣ 看默认路由

```bash
ip route
```

你会看到：

```
default via 192.168.1.1
```

这就是你的路由器。

------

## 2️⃣ 抓包看 WiFi

```bash
sudo tcpdump -i wlp4s0
```

你会看到真实流量。

### 4️⃣ /24 = 子网掩码
