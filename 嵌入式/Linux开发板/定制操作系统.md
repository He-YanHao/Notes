------

# 一、先搞清楚：一个嵌入式 Linux 系统由什么组成？

完整结构是：

```text
BootROM
  ↓
U-Boot
  ↓
Linux Kernel
  ↓
Device Tree
  ↓
Root FileSystem (Ubuntu / BusyBox)
  ↓
你的应用程序
```

你现在用的是：

- Rockchip 官方内核
- Ubuntu RootFS
- 厂商预编译好的镜像

------

# 二、定制的三种路线

------

# 🥉 路线1：在现有 Ubuntu 上“裁剪优化”（最简单）

适合：

> 想变精简，但不想重构系统

你可以：

### 1️⃣ 删除不需要的软件

```bash
sudo apt remove package_name
```

查看已安装包：

```bash
dpkg -l
```

------

### 2️⃣ 关闭不需要的服务

```bash
systemctl list-unit-files
systemctl disable 服务名
```

------

### 3️⃣ 自己编译内核（进阶一点）

Rockchip 提供 BSP。

你可以：

- 修改 Device Tree
- 关闭不用的驱动
- 编译自己的 Image

------

👉 优点：风险低
👉 缺点：系统体积仍然较大

------

# 🥈 路线2：用 Buildroot 自己生成系统（推荐）

如果你想：

- 极度精简
- 只运行你程序
- 减少存储占用
- 提高启动速度

建议用：

Buildroot

特点：

- 自动生成 rootfs
- 自动编译内核
- 可定制 BusyBox
- 生成最小系统

优点：

- 非常适合嵌入式
- 学习价值极高
- 系统只有几十 MB

缺点：

- 需要学习成本
- 不再是 Ubuntu

------

# 🥇 路线3：用 Yocto（工业级方案）

工业项目常用：

Yocto Project

特点：

- 完整发行版定制
- 可维护大规模产品
- 支持长期维护

但：

> 学习曲线很陡

不建议作为第一步。

# 四、真实定制流程（工程师路线）

## 第一步：搞清楚当前系统结构

在 RK3566 上执行：

```bash
uname -a
cat /proc/cmdline
lsblk
```

看看：

- 内核版本
- rootfs 在哪里
- 是否是 ext4

------

## 第二步：拿到 Rockchip SDK

Rockchip 官方提供：

- U-Boot
- Kernel
- 编译工具链

你要做的不是“重装系统”，而是：

> 学会自己生成镜像
