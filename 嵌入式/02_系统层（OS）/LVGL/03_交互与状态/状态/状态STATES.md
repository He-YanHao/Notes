# LVGL 状态（States）详解

状态描述对象当前所处的交互情形，可同时存在多个组合状态。

## 基础状态定义

| 状态               | 值     | 说明                             |
| ------------------ | ------ | -------------------------------- |
| LV_STATE_DEFAULT   | 0x0000 | 默认状态，控件的初始基础样式     |
| LV_STATE_CHECKED   | 0x0001 | 选中状态，如复选框、开关被勾选   |
| LV_STATE_FOCUSED   | 0x0002 | 焦点状态，通过触摸或鼠标点击获得 |
| LV_STATE_FOCUS_KEY | 0x0004 | 键盘焦点状态，通过键盘导航获得   |
| LV_STATE_EDITED    | 0x0008 | 编辑状态，如文本输入框正在编辑   |
| LV_STATE_HOVERED   | 0x0010 | 悬停状态，鼠标指针悬停在控件上   |
| LV_STATE_PRESSED   | 0x0020 | 按压状态，控件正在被按下未释放   |
| LV_STATE_SCROLLED  | 0x0040 | 滚动状态，控件内容正在滚动       |
| LV_STATE_DISABLED  | 0x0080 | 禁用状态，控件不可交互且外观灰化 |

## 用户自定义状态

| 状态            | 值     | 说明            |
| --------------- | ------ | --------------- |
| LV_STATE_USER_1 | 0x1000 | 用户自定义状态1 |
| LV_STATE_USER_2 | 0x2000 | 用户自定义状态2 |
| LV_STATE_USER_3 | 0x4000 | 用户自定义状态3 |
| LV_STATE_USER_4 | 0x8000 | 用户自定义状态4 |

## 特殊值

LV_STATE_ANY 值为 0xFFFF，用于匹配所有状态的通配符。

## 设计特点

位标志设计：每个状态值占一个独立的二进制位，可以通过位或运算进行组合。例如：

LV_STATE_DISABLED | LV_STATE_CHECKED 表示控件同时处于禁用和选中状态。

分层结构：低8位为系统预定义状态，高4位保留给应用程序自定义扩展，0x0080至0x0FFF区间预留未使用。

这种设计使控件状态管理非常灵活，既支持标准GUI交互状态，也允许开发者按需扩展自定义状态。
