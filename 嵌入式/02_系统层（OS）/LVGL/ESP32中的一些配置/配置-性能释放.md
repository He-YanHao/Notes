# 配置-性能释放

## 配置目标MCU

使用：

```
idf.py set-target esp32s3
```

为项目设置编译目标，如果不设置默认为ESP32。

使用：

```
idf.py menuconfig
```

为项目配置对应的硬件资源，

### 配置SPI flash大小

```
Serial flasher config  --->
    Flash size (16 MB)  --->
		( ) 2 MB
		( ) 4 MB
		( ) 8 MB
		(X) 16 MB
```

### 配置SPI flash访问模式：

```
Serial flasher config  --->
	Flash SPI mode (DIO)  --->
		(X) QIO        # 最快
		( ) QOUT       # 第二快
		( ) DIO        # 第三快
		( ) DOUT       # 最慢
```

>   需要硬件支持



### 配置分区表：

```
Partition Table  --->
	Partition Table (Custom partition table CSV)  --->
		(X) Custom partition table CSV
```

在项目根目录放 `partitions.csv` 文件，内容格式如下：

```csv
# ESP-IDF Partition Table
# Name,     Type,   SubType,  Offset,    Size,    Flags
# 用户参数NVS分区 (可读写 存储用户配置 WiFi配置 BLE配置等)
nvs,        data,   nvs,      0x9000,    16K,
# PHY初始化数据分区 (WiFi/BLE校准数据 固定4KB)
phy_init,   data,   phy,      0xF000,    4K,
# 用户应用分区
factory,    app,    factory,  0x20000,   6M,
# 文件系统分区
storage,    data,   littlefs, 0x620000,  9M,
```



### 配置PSRAM

需要更改或更改为：

```
Component config → ESP PSRAM → Support for external, SPI-connected RAM → SPI RAM config
	Espressif IoT Development Framework Configuration
Mode (QUAD/OCT) of SPI RAM chip in use (Octal Mode PSRAM)  --->
	( ) Quad Mode PSRAM     # 四线
	(X) Octal Mode PSRAM    # 八线
[*] Move Instructions in Flash to PSRAM (NEW)
	# 把原本存放在 Flash 中的程序指令复制到 PSRAM 中执行。
	# 如果 PSRAM 速度高于你的 Flash（比如 Octal 80 MHz PSRAM vs 40 MHz QIO Flash），可以提升代码执行速度，但消耗 PSRAM 空间。
[*] Move Read-Only Data in Flash to PSRAM (NEW)
	# 将 Flash 中的只读数据（.rodata 段）搬到 PSRAM，同样为了加速数据访问。
	Set RAM clock speed (40Mhz clock speed)  --->
    	# 和 Flash 共用时钟线需要保持一致
(1024) Maximum malloc() size, in bytes, to always put in internal memory (NEW)
	# 任何 malloc() 请求的大小 ≤ 此阈值 时，分配器会优先使用速度更快的内部 SRAM，而不是 PSRAM。
	# 这样可以避免频繁分配小对象时落入较慢的外部 RAM，提高性能。
```

>   完整介绍：
>
>   ```
>   Component config → ESP PSRAM → Support for external, SPI-connected RAM → SPI RAM config
>   	Espressif IoT Development Framework Configuration
>   Mode (QUAD/OCT) of SPI RAM chip in use (Octal Mode PSRAM)  --->
>   	( ) Quad Mode PSRAM     # 四线
>   	(X) Octal Mode PSRAM    # 八线
>   Type of SPIRAM chip in use (Auto-detect)  --->
>   	# 型号 不改 使用auto自动识别
>   [ ] Enable Executable in place from (XiP) from PSRAM feature (NEW)
>   	# 允许 CPU 直接从 PSRAM 执行指令（XIP）。
>   	# 启用后可以将部分 .text 段映射到 PSRAM，从而减少对 Flash 的读取，但需要对 Flash 和 PSRAM 进行内存映射配置，通常需要额外处理。
>   [ ] Move Instructions in Flash to PSRAM (NEW)
>   	# 把原本存放在 Flash 中的程序指令复制到 PSRAM 中执行。
>   	# 如果 PSRAM 速度高于你的 Flash（比如 Octal 80 MHz PSRAM vs 40 MHz QIO Flash），可以提升代码执行速度，但消耗 PSRAM 空间。
>   [ ] Move Read-Only Data in Flash to PSRAM (NEW)
>   	# 将 Flash 中的只读数据（.rodata 段）搬到 PSRAM，同样为了加速数据访问。
>   	Set RAM clock speed (40Mhz clock speed)  --->
>       	# 和 Flash 共用时钟线需要保持一致
>   [ ] Enable SPI RAM ECC (NEW)
>   	# 若硬件支持 则开启后消耗性能获取纠错支持
>   [*] Initialise the PSRAM related hardware (NEW)
>   [*]     Initialize PSRAM during startup, including the hardware and memory related configurations (NEW)
>   			# 勾选即可 自动处理PSRAM的初始化
>   [*] Pre-configure memory protection for PSRAM (NEW)
>   	# 为 PSRAM 区域预设内存保护
>   [ ] Ignore PSRAM when not found (NEW)
>   	# 若勾选 则缺失PSRAM不视为严重错误
>       SPI RAM access method (Make RAM allocatable using malloc() as well)  --->
>       	# 通过标准 C 的 malloc() 或 ESP-IDF 的 heap_caps_malloc(..., MALLOC_CAP_SPIRAM) 从 PSRAM 中分配内存。
>   [*] Run memory test on SPI RAM initialization (NEW)
>   	# 启动时对 PSRAM 进行简单读写测试，确保硬件正常工作。
>   (16384) Maximum malloc() size, in bytes, to always put in internal memory (NEW)
>   	# 任何 malloc() 请求的大小 ≤ 此阈值 时，分配器会优先使用速度更快的内部 SRAM，而不是 PSRAM。
>   	# 这样可以避免频繁分配小对象时落入较慢的外部 RAM，提高性能。
>   [ ] Try to allocate memories of WiFi and LWIP in SPIRAM firstly. If failed, allocate internal memory (NEW)
>   	# 如果勾选，Wi‑Fi 和 LwIP 协议栈的内存会优先从 PSRAM 分配，失败才用内部 RAM。
>   	# 一般不推荐开启，因为网络栈对时延敏感，外部 PSRAM 的访问延迟可能影响网络性能和稳定性。
>   (32768) Reserve this amount of bytes for data that specifically needs to be in DMA or internal memory (NEW)
>   	# 预留一部分内部内存（默认 32768 字节），专门给必须放在内部 RAM 中的需求使用，比如 DMA 缓冲区、Wi-Fi 协议栈的部分结构等。
>   	# 即使系统启用了 PSRAM，这部分内部 RAM 也不会被 malloc 挪用，确保关键分配不会失败。
>   [ ] Allow .bss segment placed in external memory (NEW)
>   	# 允许将 .bss 段放在 PSRAM 中。
>   	# 这会进一步释放内部 SRAM，但访问这些变量时会慢一些。
>   [ ] Allow .noinit segment placed in external memory (NEW)
>   	# 允许将 .noinit 段放在 PSRAM 中。
>   	# 这会进一步释放内部 SRAM，但访问这些变量时会慢一些。
>   ```

注意：

-   仅仅在包含 `esp_psram` 组件时可以进行配置。
-   `PSRAM` 如果和 `SPI flash` 公用时钟线，则二者频率必须一致。
-   `QUAD/OCT` 必须选择正确，选择错误程序会崩溃。

### 配置CPU主频

```
Component config  --->
	ESP System Settings  --->
		CPU frequency (240 MHz)  --->
			( ) 80 MHz
			( ) 160 MHz
			(X) 240 MHz
```

### 配置FreeRTOS

```
Component config  --->
	FreeRTOS  --->
        Kernel  --->
            (100)  configTICK_RATE_HZ
            	1000
			(1)    configTIMER_TASK_PRIORITY
            	3
            (1536) configMINIMAL_STACK_SIZE (Idle task stack size)
            	2048
            (2048) configTIMER_TASK_STACK_DEPTH
            	4096
```

## 配置LVGL

```
Component config  --->
	LVGL configuration  --->
		[ ] Uncheck this to use custom lv_conf.h
			取消勾选启用lv_conf.h 优先级高于ESP-IDF的配置
```

打开

```
#define LV_USE_PERF_MONITOR 1
#define LV_USE_MEM_MONITOR 1
```

v8.x.x版本可以直接显示内存占用和帧率和CPU占用率
