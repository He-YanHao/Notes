### 二、状态（States）— 对象当前所处的交互情形

状态描述对象**当前所处的交互情形**，可同时存在多个（组合状态）。

| 状态        | 触发条件                                 |
| ----------- | ---------------------------------------- |
| `DEFAULT`   | 正常态，未被触摸、未聚焦、未选中、未禁用 |
| `CHECKED`   | 控件被切换为选中状态                     |
| `PRESSED`   | 正被手指或鼠标按下但未释放               |
| `DISABLED`  | 被禁用，外观灰化，不可交互               |
| `FOCUSED`   | 获得输入焦点（通常通过键盘/编码器导航）  |
| `FOCUS_KEY` | 获得焦点且有按键事件发生                 |
| `EDITED`    | 处于编辑模式（如文本输入中）             |
| `SCROLLED`  | 内容发生了滚动偏移                       |

**组合状态示例**：

| 组合状态             | 触发条件           |
| -------------------- | ------------------ |
| `CHECKED \| PRESSED` | 已选中状态下被按住 |

### 三、部件（Parts）— 对象的组成部分

部件代表控件的**不同组成部分**，同一控件可以有多个部件，每个部件可独立设置样式。

| 部件            | 说明                     | 适用控件示例               |
| --------------- | ------------------------ | -------------------------- |
| `Main part`     | 主体部分（默认样式基准） | 所有控件                   |
| `Selected part` | 选中指示部分             | Checkbox、Dropdown、Roller |

> 并非所有控件都有 `Selected part`，只有存在“选中/未选中”视觉区分的控件才需要。

**使用方法**：将部件与状态组合，为特定情况设置样式。
