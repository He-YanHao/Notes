# 图层顺序Z-order

Z-order 决定同一父对象下子对象的绘制顺序。后创建的对象默认在上层。

## 一、核心概念

| 规则             | 说明                                      |
| ---------------- | ----------------------------------------- |
| 绘制顺序         | 先画底层，后画顶层，后画的在上面          |
| 同一父对象内可比 | 不同父对象的子对象之间不可直接比较Z-order |
| screen独立       | 每个screen有自己的Z-order体系             |

## 二、API函数

| 函数                   | 作用               |
| ---------------------- | ------------------ |
| lv_obj_move_foreground | 移动到最顶层       |
| lv_obj_move_background | 移动到最后层       |
| lv_obj_move_to_index   | 移动到指定索引位置 |

## 三、示例

```c
// 创建两个对象，后创建的在上面
lv_obj_t * obj1 = lv_obj_create(parent);
lv_obj_t * obj2 = lv_obj_create(parent);  // obj2 覆盖 obj1

// 将 obj1 移到最前面
lv_obj_move_foreground(obj1);  // 现在 obj1 覆盖 obj2
```

## 四、总结

| 要点     | 说明                          |
| -------- | ----------------------------- |
| 默认顺序 | 按创建顺序，后创建在上层      |
| 显式调整 | 可用API动态调整前后顺序       |
| 跨父对象 | 不同父对象之间的Z-order不可比 |
