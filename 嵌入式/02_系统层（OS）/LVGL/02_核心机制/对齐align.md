# 对齐 Align

对齐是LVGL中摆放控件位置的核心机制。与直接指定坐标不同，对齐建立的是相对位置语义。

## 一、对齐与直接定位的根本区别

| 方式                 | 说明             | 问题                               |
| -------------------- | ---------------- | ---------------------------------- |
| lv_obj_set_pos(x, y) | 直接指定绝对坐标 | 不关心父对象大小，分辨率一变就错位 |
| lv_obj_align()       | 描述相对位置关系 | LVGL自动计算坐标，适配不同分辨率   |

对齐的本质是告诉LVGL：“把我放在参照物的左上/中央/右下”，而不是给出具体数字。

## 二、对齐的三个核心问题

每次对齐操作都回答三个问题：

1. 谁要被对齐？（obj）
2. 对齐到谁？（base）
3. 对齐到哪个方位？（align type）

## 三、核心API

### 对齐到父对象（最常用）

```c
lv_obj_align(obj, LV_ALIGN_TOP_LEFT, 0, 0);
```

含义：将obj对齐到父对象的左上角，X轴偏移0像素，Y轴偏移0像素。

如果传NULL作为参照物，默认对齐到父对象（屏幕也算父对象）。

### 对齐到另一个对象

```c
lv_obj_align_to(obj, ref, LV_ALIGN_OUT_RIGHT_MID, 10, 0);
```

含义：将obj放置在ref的右侧中间位置，间隔10像素。

## 四、对齐类型分类

### 内部对齐（点在内部）

对齐的本质是两个点的重合。

| 对齐方式              | 含义                                |
| --------------------- | ----------------------------------- |
| LV_ALIGN_TOP_LEFT     | obj的左上角对齐到参照物的左上角     |
| LV_ALIGN_TOP_MID      | obj的顶部中点对齐到参照物的顶部中点 |
| LV_ALIGN_TOP_RIGHT    | obj的右上角对齐到参照物的右上角     |
| LV_ALIGN_LEFT_MID     | obj的左中點对齐到参照物的左中点     |
| LV_ALIGN_CENTER       | obj的中心点对齐到参照物的中心点     |
| LV_ALIGN_RIGHT_MID    | obj的右中点对齐到参照物的右中点     |
| LV_ALIGN_BOTTOM_LEFT  | obj的右下角对齐到参照物的右下角     |
| LV_ALIGN_BOTTOM_MID   | obj的底部中点对齐到参照物的底部中点 |
| LV_ALIGN_BOTTOM_RIGHT | obj的右下角对齐到参照物的右下角     |

### 外部对齐（点在外部）

OUT系列用于将对象放置在参照物外部，不产生重叠。

| 对齐方式                  | 含义             |
| ------------------------- | ---------------- |
| LV_ALIGN_OUT_TOP_LEFT     | 放在参照物左上方 |
| LV_ALIGN_OUT_TOP_MID      | 放在参照物正上方 |
| LV_ALIGN_OUT_TOP_RIGHT    | 放在参照物右上方 |
| LV_ALIGN_OUT_LEFT_MID     | 放在参照物左侧   |
| LV_ALIGN_OUT_RIGHT_MID    | 放在参照物右侧   |
| LV_ALIGN_OUT_BOTTOM_LEFT  | 放在参照物左下方 |
| LV_ALIGN_OUT_BOTTOM_MID   | 放在参照物正下方 |
| LV_ALIGN_OUT_BOTTOM_RIGHT | 放在参照物右下方 |

OUT的含义就是不重叠，对象被放置在参照物的边界之外。

## 五、对齐是一次性操作

对齐只在调用时计算一次坐标。如果父对象尺寸改变或屏幕分辨率变化，不会自动重新对齐。

解决方案：
- 方法一：重新调用lv_obj_align
- 方法二：使用Flex或Grid布局（持续管理布局）

## 六、对齐与其他机制的关系

| 机制    | 关系                                         |
| ------- | -------------------------------------------- |
| Z-order | 无关。对齐决定位置，Z-order决定谁在上面      |
| Style   | 无关。Style影响外观和padding，对齐只关心外框 |
| 子对象  | 不影响。对齐只改变当前对象位置               |

## 七、常见误区

| 误区             | 正确理解                         |
| ---------------- | -------------------------------- |
| 对齐会一直生效   | 不会，它是一次性计算             |
| 对齐会影响子对象 | 不会，只影响当前对象             |
| 对齐就是布局     | 不是，对齐是摆放，布局是持续管理 |

## 八、工程建议

- 临时摆放位置用align
- 复杂界面用flex或grid布局
- OUT系列是最常用的外部对齐方式
- 不要在循环中频繁调用align
- 分辨率可变时不要依赖align进行整体布局

## 九、示例

```c
lv_obj_t * a = lv_obj_create(scr);
lv_obj_set_size(a, 40, 40);

lv_obj_t * b = lv_obj_create(scr);
lv_obj_set_size(b, 20, 20);

lv_obj_align(a, LV_ALIGN_CENTER, 0, 0);
lv_obj_align_to(b, a, LV_ALIGN_OUT_RIGHT_MID, 5, 0);
```

效果：a位于屏幕中央，b紧贴在a右侧间隔5像素。

## 十、总结

对齐不是算坐标，而是建立相对位置语义。

下一步进阶方向：Flex布局（真正不用算坐标）。
