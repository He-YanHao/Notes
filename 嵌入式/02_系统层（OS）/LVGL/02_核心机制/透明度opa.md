# 透明度 Opa

LVGL的透明度不是一个直觉系统，而是一个分层多通道混合模型。透明度不是颜色的一个属性，而是由多个层级共同决定：颜色本身、Style的透明度、对象本身的透明度、父对象裁剪。

## 一、透明度单位

透明度统一使用 `lv_opa_t` 类型，本质是 `uint8_t`。

| 定义          | 数值 | 含义        |
| ------------- | ---- | ----------- |
| LV_OPA_TRANSP | 0    | 完全透明    |
| LV_OPA_0      | 0    | 完全透明    |
| LV_OPA_10     | 25   | 约10%不透明 |
| LV_OPA_20     | 51   | 约20%不透明 |
| LV_OPA_30     | 76   | 约30%不透明 |
| LV_OPA_40     | 102  | 约40%不透明 |
| LV_OPA_50     | 128  | 半透明      |
| LV_OPA_60     | 153  | 约60%不透明 |
| LV_OPA_70     | 179  | 约70%不透明 |
| LV_OPA_80     | 204  | 约80%不透明 |
| LV_OPA_90     | 230  | 约90%不透明 |
| LV_OPA_100    | 255  | 完全不透明  |
| LV_OPA_COVER  | 255  | 完全不透明  |

重要提示：透明度不是0到1的小数，而是0到255的整数。

## 二、颜色本身没有透明度

LVGL的 `lv_color_t` 本身不包含alpha通道。例如：

```c
lv_color_t c = lv_color_hex(0xFF0000);
```

这里只有RGB（或RGB565/RGB888），没有alpha通道。透明度不是颜色的一部分，而是由样式单独控制。

## 三、透明度来自Style

### 样式级透明度属性

| 属性                       | 作用               |
| -------------------------- | ------------------ |
| LV_STYLE_BG_OPA            | 背景透明度         |
| LV_STYLE_BORDER_OPA        | 边框透明度         |
| LV_STYLE_TEXT_OPA          | 文本透明度         |
| LV_STYLE_IMAGE_RECOLOR_OPA | 图像重新着色透明度 |
| LV_STYLE_LINE_OPA          | 线条透明度         |
| LV_STYLE_ARC_OPA           | 圆弧透明度         |

### 示例：半透明红色背景

```c
lv_obj_set_style_bg_color(obj, lv_color_hex(0xFF0000), 0);
lv_obj_set_style_bg_opa(obj, LV_OPA_50, 0);
```

颜色不带透明，透明度单独指定。

## 四、对象整体透明度

`LV_STYLE_OPA` 作用于整个对象最终像素，再乘一个透明度，包含背景、边框、子对象（会传递）。

```c
lv_obj_set_style_opa(obj, LV_OPA_50, 0);
```

### 透明度级联相乘

透明度会沿对象树级联相乘。如果父对象透明度为50%，子对象透明度为50%，则最终透明度为0.5 × 0.5 = 0.25。

| 层级   | 透明度 | 累计效果         |
| ------ | ------ | ---------------- |
| 父对象 | 50%    | 0.5              |
| 子对象 | 50%    | 0.5 × 0.5 = 0.25 |

## 五、父对象如何影响子对象

### 透明度继承

最终像素透明度 = 父对象透明度 × 子对象透明度 × 样式透明度。

### 裁剪

如果父对象开启裁剪：
```c
lv_obj_set_style_clip_corner(parent, true, 0);
```
子对象绘制在父区域外的部分直接被裁掉，与透明度无关，但视觉效果常被误认为透明。

## 六、RGB565屏幕的半透明支持

即使屏幕硬件不支持alpha通道，LVGL也能实现半透明效果。

渲染流程：
1. 前景颜色（RGB）
2. 背景颜色（RGB）
3. 结合alpha值
4. 软件混合计算
5. 结果量化成RGB565
6. 写入framebuffer

结论：即使屏幕不支持alpha通道，视觉上的透明依然成立，但会消耗更多CPU进行混合计算。

## 七、常见透明问题排查

| 现象             | 可能原因                         |
| ---------------- | -------------------------------- |
| 设置颜色透明无效 | 忘了设置bg_opa，颜色本身不带透明 |
| 子对象不显示     | 父对象opa = 0                    |
| 层叠后颜色发灰   | 多级半透明叠加 + RGB565量化误差  |
| 透明区域显示异常 | 裁剪误认为透明                   |
| 性能下降         | 大量透明混合操作                 |

## 八、调透明度实用法则

一次只改一个层级，按以下顺序调试：

1. bg_color（背景颜色是否正确）
2. bg_opa（背景透明度是否设置）
3. opa（对象整体透明度）
4. 父对象是否有opa

## 九、与Z-order的关系

透明度和Z-order是两个独立系统：
- 透明度决定颜色混合方式
- Z-order决定绘制顺序
- 两者共同影响最终显示效果

## 十、总结

| 要点               | 说明             |
| ------------------ | ---------------- |
| 颜色本身没有alpha  | 透明度来自Style  |
| 透明度范围0-255    | 使用lv_opa_t类型 |
| 透明度级联相乘     | 父×子×样式       |
| RGB565支持半透明   | 软件混合后输出   |
| 透明度≠裁剪        | 两个独立机制     |
| 多级透明有性能代价 | 按需使用         |

理解透明度模型是从使用LVGL走向掌握其渲染引擎的关键一步。
