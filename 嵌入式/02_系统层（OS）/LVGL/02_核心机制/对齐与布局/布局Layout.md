# 布局 Layout

布局是 LVGL 中自动排列子对象位置的机制。与手动指定坐标不同，布局会根据容器大小和预设规则自动计算子对象位置，当容器大小改变时自动重新排列。

## 布局与手动定位的区别

手动定位使用 lv_obj_set_pos 指定绝对坐标，不关心父对象大小，分辨率一变就容易错位。

布局使用容器自动排列子对象，当容器大小改变时子对象会自动重新排列，无需手动计算。适用于复杂界面和响应式设计。

## 布局类型

LVGL 支持三种布局类型：

- NONE：无布局，手动定位，适合少量固定位置的控件
- FLEX：弹性布局，单行或单列排列，适合工具栏、列表、按钮组
- GRID：网格布局，多行多列表格排列，适合仪表盘、键盘、表单

## 布局的通用属性

所有布局都支持以下样式属性：

内边距 pad_top、pad_bottom、pad_left、pad_right 控制容器内容区域与边缘的距离。

间距 pad_row 控制行与行之间的间距，pad_column 控制列与列之间的间距。

## 忽略布局

子对象可以设置 IGNORE_LAYOUT 标志使其脱离布局管理。设置后该子对象被布局系统完全忽略，保留手动设置的坐标，不占用布局槽位，其他子对象会像它不存在一样进行布局。适用于浮动按钮、徽章或绝对定位的装饰元素。

## 布局选择指南

垂直排列的菜单或列表适合使用 Flex Column。水平排列的按钮组适合使用 Flex Row。工具栏图标适合使用 Flex Row Space Evenly。表单布局如标签加输入框的组合适合使用 Grid。仪表盘卡片适合使用 Grid。键盘适合使用 Grid。单行滚动区域适合使用 Flex Row Wrap。需要绝对定位的装饰元素使用 IGNORE_LAYOUT。

## 总结

布局是自动排列机制，无需手动计算坐标。Flex 用于单行或单列排列，方向和对齐灵活。Grid 用于多行多列排列，适合复杂表格布局。不同布局可以混合使用，容器可以嵌套不同布局。IGNORE_LAYOUT 让特定控件脱离布局。
