# 布局 Grid

Grid 是一种基于二维单元格的布局，将子元素放置在明确的行和列中，类似于 CSS Grid。当内容需要在两个轴上对齐时，例如仪表盘或表单，可以使用此布局。

## 网格结构定义

网格的行数和列数由描述符数组决定。数组中有几个有效值，就有几行或几列。数组的最后一个元素必须是 `LV_GRID_TEMPLATE_LAST`，标记数组结束。

列描述符定义每一列的宽度，行描述符定义每一行的高度。

## 列宽与行高的类型

固定数值使用像素作为单位，列宽或行高为固定值。

LV_GRID_CONTENT 表示尺寸根据子对象内容自动调整。

LV_GRID_FR(x) 表示按比例分配剩余空间，x 为权重值。

## 网格对齐方式

Grid column align 控制列组在容器水平方向上的排列位置。当所有列的总宽度小于容器宽度时，此属性决定列组是靠左、居中、靠右还是拉伸填满。

START 使列组靠容器左侧排列，右侧留有空白。CENTER 使列组在容器水平方向上居中排列，左右两侧留有相等空白。END 使列组靠容器右侧排列，左侧留有空白。STRETCH 使列组拉伸填满整个容器宽度，剩余空间按比例分配到各列。

Grid row align 控制行组在容器垂直方向上的排列位置，逻辑与列对齐类似。

## 子对象在网格中的位置

每个子对象需要声明自己所在的行和列。起始行列索引从 0 开始，占据行列数最小为 1。

子对象还可以设置在单元格内的对齐方式。X align 控制水平对齐，Y align 控制垂直对齐。可选值包括 START、CENTER、END、STRETCH。

## 忽略布局

设置 ignore_layout 为 true 后，该子元素被 Grid 布局完全忽略。它保留手动设置的坐标，不占用布局槽位，其他子元素会像它不存在一样进行布局。适用于浮动按钮、徽章或绝对定位的装饰元素。

## 从右到左布局

设置容器的 base_dir 为 RTL 可以反转列顺序。网格声明的列顺序不变，但单元格 0 会出现在右侧。适用于从右到左的语言或镜像布局。

## 子网格

可以将一个 Grid 嵌套在另一个 Grid 的单元格内。内层网格可以省略自己的行列描述符，继承父网格的轨道尺寸，使其单元格与外层网格的列和行保持对齐。

## 工程建议

需要多行多列表格排列时使用 Grid 而非 Flex。等比例分配空间时使用 FR 单位而非固定像素。需要跨行跨列时使用单元格跨度属性。浮动元素使用 ignore_layout 脱离布局管理。RTL 布局使用 base_dir 而非手动调整顺序。
