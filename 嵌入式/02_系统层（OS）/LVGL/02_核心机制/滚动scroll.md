# 滚动 Scroll

滚动是LVGL中处理内容超出父容器时的行为机制。理解滚动需要区分两个概念：滚动能力（行为）和滚动条（视觉）。

## 一、核心结论

LVGL的默认滚动行为取决于父对象是否具备滚动能力：

- 父对象可滚动 → 子对象超出 → 出现滚动行为
- 父对象不可滚动 → 子对象超出 → 被裁剪（不显示）

重要提示：是否显示滚动条，取决于父对象是否开启了滚动能力，而不是是否有子对象超界。

## 二、两个独立的机制

| 机制        | 作用                     |
| ----------- | ------------------------ |
| 裁剪 clip   | 决定超出部分是否显示     |
| 滚动 scroll | 决定能否滑动查看超出内容 |

这两个机制相互独立，很多混淆源于将它们混为一谈。

## 三、默认滚动能力

普通对象 `lv_obj_create()` 在LVGL中默认是可滚动的。这意味着：
- 子对象超出父容器
- 父对象内容尺寸自动扩大
- 用户可以滑动查看超出内容

## 四、超出不显示的原因

以下几种情况会导致超出内容不可见：

### 情况A：关闭滚动能力

```c
lv_obj_clear_flag(parent, LV_OBJ_FLAG_SCROLLABLE);
```

效果：子对象仍然存在，但绘制被裁剪，超出部分不显示，没有滚动条。这是最常见的原因。

### 情况B：开启裁剪

```c
lv_obj_set_style_clip_corner(parent, true, 0);
```

效果：子对象超出父容器区域的部分被直接裁掉，与滚动无关。

### 情况C：父对象不是容器类型

例如 lv_btn、lv_label、lv_img 等对象，设计目的不是作为滚动容器，即使有子对象也表现为裁剪而非滚动。

## 五、滚动条与滚动能力

滚动条只是视觉提示，滚动能力是行为属性，两者独立。

滚动条模式通过以下API设置：

```c
lv_obj_set_scrollbar_mode(parent, LV_SCROLLBAR_MODE_AUTO);
```

| 模式                     | 含义                   |
| ------------------------ | ---------------------- |
| LV_SCROLLBAR_MODE_OFF    | 永不显示滚动条         |
| LV_SCROLLBAR_MODE_ON     | 始终显示滚动条         |
| LV_SCROLLBAR_MODE_AUTO   | 内容超出时自动显示     |
| LV_SCROLLBAR_MODE_ACTIVE | 滚动时显示，闲置后隐藏 |

默认情况下，父对象可以滚动，但滚动条可能是隐藏的（AUTO模式）。

## 六、对比实验

### 情况1：默认行为（可滚动）

```c
lv_obj_t * parent = lv_obj_create(scr);
lv_obj_set_size(parent, 100, 100);

lv_obj_t * child = lv_obj_create(parent);
lv_obj_set_size(child, 100, 200);
```

结果：parent可滚动，child超出部分可通过滑动查看。

### 情况2：关闭滚动（被裁剪）

```c
lv_obj_clear_flag(parent, LV_OBJ_FLAG_SCROLLABLE);
```

结果：child超出部分不显示，无法滚动。

## 七、问题判断三步法

当滚动行为不符合预期时，按以下顺序检查：

1. 父对象是否拥有 `LV_OBJ_FLAG_SCROLLABLE` 标志？
2. 是否启用了裁剪（clip corner）？
3. 父对象类型是否设计为容器？

## 八、准确的说法

常见的“子对象超出 → 自动有滚动条”说法是不完整的。

准确描述应为：当父对象是可滚动对象，且内容尺寸超出时，LVGL会启用滚动机制，并根据滚动条模式决定是否显示滚动条。

## 九、总结

| 要点                          | 说明                 |
| ----------------------------- | -------------------- |
| 子对象超出 ≠ 一定有滚动条     | 滚动能力是前提       |
| 滚动是行为，滚动条是外观      | 两者独立控制         |
| 默认 lv_obj_create 是可滚动的 | 普通容器默认支持滚动 |
| 关闭 scrollable 会裁剪        | 超出内容不可见       |
| clip corner 会强制裁剪        | 与滚动无关           |

理解滚动是行为、滚动条是外观这一区分，是从运行demo走向编写稳定UI的关键一步。
