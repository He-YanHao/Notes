```bash
git branch -r
```

获取版本分支，例如：

```bash

```

git checkout -b release/v8 origin/release/v8

## 在 lv_port_pc_vscode 中使用 LVGL 8.4.0 的完整步骤

### 环境准备
- 已安装 Git、GCC、Make、SDL2 开发库
- 系统：Linux / macOS / Windows (Git Bash + MinGW)

---

### 1. 克隆仓库并进入
```bash
git clone https://github.com/lvgl/lv_port_pc_vscode.git
cd lv_port_pc_vscode
```

### 2. 切换到干净的 v8 分支
由于远程的 `release/v8` 分支已被机器人自动提交污染，我们选用该分支上最后一个人工提交 `b45aef7`，它对应真正的 v8 版本。

```bash
git checkout b45aef7
git submodule update --init --recursive
```

### 3. 确认当前 LVGL 版本（应该是 8.3.x）
```bash
cd lvgl
head -20 lvgl.h | grep VERSION
```
输出示例：
```
#define LVGL_VERSION_MAJOR 8
#define LVGL_VERSION_MINOR 3
#define LVGL_VERSION_PATCH 10
```

### 4. 升级 lvgl 子模块到 v8.4.0
```bash
git fetch --tags
git checkout v8.4.0
```
再次确认版本：
```bash
head -20 lvgl.h | grep VERSION
```
应显示 8.4.0。

### 5. 在父仓库中记录子模块变更
```bash
cd ..
git add lvgl
git commit -m "upgrade lvgl to v8.4.0"
```

### 6. 清理无关文件
旧分支可能包含不完整的 FreeRTOS 目录和编译产物，需要删除：
```bash
rm -rf FreeRTOS bin
```

### 7. 编译（使用 Makefile）
```bash
make clean
make -j$(nproc)
```
编译成功后会在 `build/bin/` 下生成 `demo` 可执行文件。

### 8. 运行模拟器
```bash
./build/bin/demo
```
出现 LVGL 窗口即表示成功。

## 常见问题

### Q1: 提示找不到 `demo` 命令
可执行文件位于 `build/bin/demo`，使用相对路径运行。

### Q2: 右下角 CPU 百分比表示什么？
这是 LVGL 内部帧率计时器算出的“帧时间占比”，不是系统全局 CPU 占用。可在 `lv_conf.h` 中将 `LV_USE_PERF_MONITOR` 设为 `0` 关闭。

### Q3: 编译时出现 FreeRTOS 相关错误
说明仓库根目录残留了 FreeRTOS 目录，直接 `rm -rf FreeRTOS` 即可。
