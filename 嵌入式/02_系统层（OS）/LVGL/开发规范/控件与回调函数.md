在 LVGL 中，这两种方式都可以实现，但**推荐使用一个回调函数，在函数内部根据事件类型（`lv_event_get_code`）进行区分处理**。这样代码更集中、易于维护，也避免了多个回调之间的状态共享问题。

---

## 方案对比

| 方式                    | 优点                                                       | 缺点                                                         |
| :---------------------- | :--------------------------------------------------------- | :----------------------------------------------------------- |
| **一个回调 + 事件判断** | 逻辑集中，状态变量（如计时器、计数）易于管理；代码可读性高 | 回调函数内分支稍多                                           |
| **三个回调分别绑定**    | 每个回调只处理单一事件，职责单一                           | 需要共享状态（如用于区分双击的计数），处理复杂；增加代码分散度 |

---

## 推荐实现：一个回调 + 事件判断

LVGL 提供了完整的事件类型用于检测点按、长按和双击：

```c
static void btn_event_cb(lv_event_t *e) {
    lv_event_code_t code = lv_event_get_code(e);
    
    switch(code) {
        case LV_EVENT_CLICKED:
            // 点按（短按并释放）
            LV_LOG_USER("点按");
            break;
            
        case LV_EVENT_LONG_PRESSED:
            // 长按（持续按压达到阈值，未释放）
            LV_LOG_USER("长按");
            break;
            
        case LV_EVENT_DOUBLE_CLICKED:
            // 双击（两次快速点按）
            LV_LOG_USER("双击");
            break;
            
        default:
            break;
    }
}

// 创建按钮并绑定回调
lv_obj_t *btn = lv_btn_create(lv_scr_act());
lv_obj_add_event_cb(btn, btn_event_cb, LV_EVENT_ALL, NULL);  // 监听所有事件
```

---

## 如果需要更精细的控制

例如，区分“长按后释放”或“双击的第二下”，可以使用更具体的事件：

| 事件类型                        | 触发时机       |
| :------------------------------ | :------------- |
| `LV_EVENT_CLICKED`              | 短按并释放     |
| `LV_EVENT_LONG_PRESSED`         | 长按（未释放） |
| `LV_EVENT_LONG_PRESSED_RELEASE` | 长按后释放     |
| `LV_EVENT_DOUBLE_CLICKED`       | 双击           |
| `LV_EVENT_PRESSED`              | 刚按下瞬间     |

---

## 如果需要自定义长按时间或双击间隔

LVGL 允许配置输入设备的默认参数：

```c
// 在初始化 LVGL 后修改
lv_indev_t *indev = lv_indev_get_act();
lv_indev_set_long_press_time(indev, 400);      // 长按阈值 400ms
lv_indev_set_long_press_repeat_time(indev, 100); // 长按重复间隔
// 双击间隔可通过 lv_indev_set_double_tap_limit(indev, 300); // 默认 300ms
```

## 三种效果共存时的注意事项

1. **事件冲突**：一次点击可能会同时触发 `LV_EVENT_CLICKED` 和 `LV_EVENT_DOUBLE_CLICKED`。LVGL 内部会延迟判断，双击时不会额外触发单击。你无需额外处理。
2. **状态变量**：如果需要在回调中区分“长按后释放”和“短按释放”，可以使用 `code` 直接区分，无需自己加标志。
3. **资源消耗**：单个回调比多个回调更省内存（少创建几个回调结构）。

---

## 总结

**绑定一个回调函数，在内部根据 `lv_event_get_code(e)` 判断事件类型**，是 LVGL 官方推荐的做法，也是实际项目中最常用的模式。
