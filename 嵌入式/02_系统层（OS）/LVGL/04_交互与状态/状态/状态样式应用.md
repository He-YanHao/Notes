# 状态样式应用

状态样式是LVGL中实现交互反馈的核心机制。控件在不同交互状态下自动切换样式，开发者无需手动管理状态，只需为每种状态定义对应的外观即可。

## 一、核心概念：谁在管理状态？

| 角色      | 职责                                                     |
| --------- | -------------------------------------------------------- |
| LVGL 内核 | 自动检测触摸、焦点、滚动等交互，改变控件的状态位         |
| 开发者    | 为不同状态配置样式，定义"按下时变蓝"、"禁用时变灰"等规则 |
| 控件      | 根据当前状态位，自动应用对应的样式                       |

你不需要写代码去"设置控件为按下状态"，LVGL会在用户触摸时自动完成。

## 二、状态优先级

当多个状态同时存在时（例如一个按钮既被按下又被选中），LVGL按以下优先级选择使用哪个样式：

| 优先级    | 状态                    | 典型场景                           |
| --------- | ----------------------- | ---------------------------------- |
| 1（最高） | `DISABLED`              | 控件不可用，任何其他状态都不应覆盖 |
| 2         | `PRESSED`               | 用户正在点击，需要立即给出视觉反馈 |
| 3         | `CHECKED`               | 开关已打开，选中状态需要明确标识   |
| 4         | `FOCUSED` / `FOCUS_KEY` | 键盘导航时的高亮边框               |
| 5         | `HOVERED`               | 鼠标悬停（触摸屏无效）             |
| 6         | `EDITED`                | 文本输入中                         |
| 7         | `SCROLLED`              | 滚动进行中                         |
| 8（最低） | `DEFAULT`               | 无任何交互时的常态外观             |

**实际意义**：如果一个按钮处于 `DISABLED` 状态，即使它同时满足 `PRESSED` 条件，也不会显示按下样式。禁用状态的优先级最高。

## 三、完整示例：按钮四种核心状态

```c
static lv_style_t style_default;
static lv_style_t style_focused;
static lv_style_t style_pressed;
static lv_style_t style_disabled;

void init_button_styles(void)
{
    // 1. 默认状态：灰色背景、圆角
    lv_style_init(&style_default);
    lv_style_set_bg_color(&style_default, lv_color_hex(0x808080));
    lv_style_set_radius(&style_default, 8);
    
    // 2. 焦点状态：蓝色边框（键盘导航时可见）
    lv_style_init(&style_focused);
    lv_style_set_border_color(&style_focused, lv_color_hex(0x2196F3));
    lv_style_set_border_width(&style_focused, 3);
    
    // 3. 按下状态：颜色加深 + 轻微缩小
    lv_style_init(&style_pressed);
    lv_style_set_bg_color(&style_pressed, lv_color_hex(0x404040));
    lv_style_set_transform_scale(&style_pressed, 245);  // 缩小约4%
    
    // 4. 禁用状态：半透明 + 文字变灰
    lv_style_init(&style_disabled);
    lv_style_set_bg_opa(&style_disabled, LV_OPA_50);
    lv_style_set_text_color(&style_disabled, lv_color_hex(0x808080));
}

void create_button(lv_obj_t * parent)
{
    lv_obj_t * btn = lv_btn_create(parent);
    
    lv_obj_add_style(btn, &style_default, LV_STATE_DEFAULT);
    lv_obj_add_style(btn, &style_focused, LV_STATE_FOCUSED);
    lv_obj_add_style(btn, &style_pressed, LV_STATE_PRESSED);
    lv_obj_add_style(btn, &style_disabled, LV_STATE_DISABLED);
    
    // 标签的样式也会随按钮状态变化
    lv_obj_t * label = lv_label_create(btn);
    lv_label_set_text(label, "Button");
    lv_obj_center(label);
    
    // 临时禁用按钮演示效果（实际场景中根据条件控制）
    // lv_obj_add_state(btn, LV_STATE_DISABLED);
}
```

## 四、组合状态示例：选中且按下

某些控件（如开关、复选框）存在 `CHECKED` 状态。当选中状态下又被按下时，可以设置专门的样式：

```c
static lv_style_t style_checked;
static lv_style_t style_checked_pressed;

void init_switch_styles(void)
{
    // 选中状态：绿色背景
    lv_style_init(&style_checked);
    lv_style_set_bg_color(&style_checked, lv_color_hex(0x4CAF50));
    
    // 选中且按下：深绿色背景 + 缩小
    lv_style_init(&style_checked_pressed);
    lv_style_set_bg_color(&style_checked_pressed, lv_color_hex(0x2E7D32));
    lv_style_set_transform_scale(&style_checked_pressed, 245);
}

void create_switch(lv_obj_t * parent)
{
    lv_obj_t * sw = lv_switch_create(parent);
    
    lv_obj_add_style(sw, &style_checked, LV_STATE_CHECKED);
    lv_obj_add_style(sw, &style_checked_pressed, LV_STATE_CHECKED | LV_STATE_PRESSED);
}
```

## 五、部件（Part）与状态的配合

样式可以同时指定作用于控件的哪个部分和哪种状态：

```c
// 滑动条的指示器部件，在按下状态时的样式
lv_obj_add_style(slider, &style_indicator_pressed, LV_PART_INDICATOR | LV_STATE_PRESSED);

// 滚动条部件，在滚动状态时的样式
lv_obj_add_style(cont, &style_scrollbar_scrolling, LV_PART_SCROLLBAR | LV_STATE_SCROLLED);
```

| 组合                                   | 含义             |
| -------------------------------------- | ---------------- |
| `LV_PART_MAIN | LV_STATE_PRESSED`      | 主体的按下样式   |
| `LV_PART_INDICATOR | LV_STATE_CHECKED` | 指示器的选中样式 |
| `LV_PART_KNOB | LV_STATE_DISABLED`     | 旋钮的禁用样式   |

## 六、状态切换的过渡动画

为了让状态切换更平滑，可以添加过渡效果：

```c
static const lv_style_prop_t props[] = {
    LV_STYLE_BG_COLOR,
    LV_STYLE_TRANSFORM_SCALE,
    0
};

static lv_style_transition_dsc_t trans;
lv_style_transition_dsc_init(&trans, props, lv_anim_path_ease_out, 150, 0, NULL);

// 将过渡应用到样式
lv_style_set_transition(&style_default, &trans);
lv_style_set_transition(&style_pressed, &trans);
```

效果：按下时背景色和大小变化会平滑过渡，而非瞬间跳变。

## 七、检测与修改状态

```c
// 检测当前状态
bool is_pressed = lv_obj_has_state(obj, LV_STATE_PRESSED);
bool is_disabled = lv_obj_has_state(obj, LV_STATE_DISABLED);

// 手动添加/移除状态（通常用于 CHECKED 和 DISABLED）
lv_obj_add_state(btn, LV_STATE_CHECKED);      // 切换到选中状态
lv_obj_remove_state(btn, LV_STATE_PRESSED);   // 清除按下状态
```

注意：`PRESSED`、`FOCUSED` 等状态由LVGL自动管理，通常不需要手动修改。开发者主要控制的是 `CHECKED` 和 `DISABLED`。

## 八、常见错误与解决

| 错误现象       | 可能原因                           | 解决方法                                           |
| -------------- | ---------------------------------- | -------------------------------------------------- |
| 按下没有反馈   | 没有为 `LV_STATE_PRESSED` 添加样式 | 添加 pressed 样式                                  |
| 禁用后仍可点击 | 没有设置 `LV_STATE_DISABLED` 状态  | 调用 `lv_obj_add_state(obj, LV_STATE_DISABLED)`    |
| 选中样式不生效 | 忘了控件本身支持 `CHECKED` 状态    | 添加 `lv_obj_add_flag(obj, LV_OBJ_FLAG_CHECKABLE)` |
| 焦点样式不显示 | 触摸屏没有焦点概念                 | 用键盘或编码器测试，或忽略焦点样式                 |
| 状态样式被覆盖 | 优先级高的状态定义了冲突属性       | 检查优先级表，或删除高优先级样式的冲突属性         |

## 九、核心要点

| 要点         | 说明                                             |
| ------------ | ------------------------------------------------ |
| 状态自动管理 | LVGL内核负责切换状态，开发者只需定义样式         |
| 优先级规则   | DISABLED > PRESSED > CHECKED > FOCUSED > DEFAULT |
| 组合使用     | 状态和部件可组合，实现精细控制                   |
| 过渡动画     | 让状态切换更平滑，提升用户体验                   |
| 触摸屏优化   | 纯触摸屏可忽略 FOCUSED/HOVERED 状态              |

理解状态、部件和样式的三层关系，才能为控件的每种交互形态设计出精准、流畅的视觉反馈。
