# RS232

这是老式串口标准。

电压非常特殊：

| 状态 | 电压       |
| ---- | ---------- |
| 1    | -3V ~ -15V |
| 0    | +3V ~ +15V |

⚠ 和TTL完全不兼容

必须使用芯片转换，比如：

-   MAX232

特点：

-   传输距离可达15米
-   抗干扰比TTL强
-   现在PC串口基本淘汰

一步步来。

------

# 一、RS232 是什么？

RS232 是一种：

>   单端串行通信物理层标准

标准来自：

-   Electronic Industries Alliance
-   标准号 EIA-232

它只定义：

-   电压范围
-   信号方向
-   接口引脚功能

它不定义数据格式（数据通常是 UART）。

------

# 二、RS232 电气定义（非常关键）

RS232 的电压和 TTL 完全不同：

| 逻辑状态        | 电压范围   |
| --------------- | ---------- |
| 逻辑 1（Mark）  | -3V ~ -15V |
| 逻辑 0（Space） | +3V ~ +15V |

⚠ 特点：

-   电压是正负电压
-   逻辑是反相的
-   中间 -3V ~ +3V 属于不确定区

------

## 为什么用正负电压？

优点：

-   抗干扰能力比TTL强
-   适合较长距离（约15米）
-   能容忍一定地电位差

------

# 三、RS232 常见芯片

MCU 是 3.3V TTL，不能直接接 RS232。

必须用电平转换芯片，例如：

-   MAX232

它的作用：

```text
TTL UART ↔ RS232 电平转换
```

------

# 四、RS232 信号线定义

RS232 不止 TX / RX。

它设计之初是为“调制解调器通信”，
所以有很多控制线。

------

## 1️⃣ 最基础三线（最常用）

| 信号 | 全称          | 方向 | 作用     |
| ---- | ------------- | ---- | -------- |
| TXD  | Transmit Data | 输出 | 发送数据 |
| RXD  | Receive Data  | 输入 | 接收数据 |
| GND  | Signal Ground | -    | 地       |

大部分嵌入式应用只用这三根。

------

## 2️⃣ 完整控制线（了解即可）

| 信号 | 作用           |
| ---- | -------------- |
| RTS  | 请求发送       |
| CTS  | 允许发送       |
| DTR  | 数据终端准备好 |
| DSR  | 数据设备准备好 |
| DCD  | 载波检测       |
| RI   | 振铃指示       |

这些用于：

-   硬件流控
-   调制解调器控制

现在嵌入式很少用。

------

# 五、DB9 接口定义（最常见）

RS232 最常见物理接口是：

>   DB9 接头（D型9针）

### 电脑端（DTE）定义：

| 引脚 | 信号 |
| ---- | ---- |
| 2    | RXD  |
| 3    | TXD  |
| 5    | GND  |
| 7    | RTS  |
| 8    | CTS  |

⚠ 注意：

-   PC 是 DTE
-   设备可能是 DCE
-   有“直连线”和“交叉线”区别

------

# 六、直连线 vs 交叉线

## 直连线

TX → TX
RX → RX

用于：

DTE ↔ DCE（电脑 ↔ 调制解调器）

------

## 交叉线（Null Modem）

TX → RX
RX → TX

用于：

DTE ↔ DTE（电脑 ↔ 电脑）

------

# 七、RS232 完整结构图

```text
应用协议
   ↓
UART
   ↓
MAX232 电平转换
   ↓
RS232 电缆
   ↓
DB9 接口
```

------

# 八、RS232 和 RS485 的核心区别

| 项目     | RS232 | RS485    |
| -------- | ----- | -------- |
| 信号方式 | 单端  | 差分     |
| 电压     | ±电压 | 差分电压 |
| 节点数   | 1对1  | 多节点   |
| 距离     | ~15m  | ~1200m   |
| 抗干扰   | 中等  | 很强     |

------

# 九、工程注意点

1️⃣ 不能直接接 MCU
2️⃣ 一定用 MAX232 类芯片
3️⃣ TX / RX 要交叉
4️⃣ GND 必须共地

------

TTL 或 RS485 更常见。

你记得得非常准确 👍

你说的这种：

>   “有专门表示可以发送 / 暂停发送 的信号线”

就是 **串口的硬件流控（Hardware Flow Control）**。

它通常出现在：

>   **UART + RS232 接口**

------

# 一、它叫什么？

最典型的是：

## RTS / CTS 硬件流控

信号名称：

-   **RTS** – Request To Send（请求发送）
-   **CTS** – Clear To Send（允许发送）

它们定义在：

Electronic Industries Alliance 制定的 RS-232 标准中。

------

# 二、它解决什么问题？

串口通信本质上是：

```text
A 一直发 → B 一直接
```

但如果：

-   波特率太高
-   MCU处理不过来
-   FIFO满了

怎么办？

👉 就需要“暂停机制”。

------

# 三、工作原理（核心逻辑）

假设 A 发给 B：

1️⃣ A 想发数据 → 拉低 RTS
2️⃣ B 如果准备好了 → 拉低 CTS
3️⃣ A 检测到 CTS 有效 → 才开始发
4️⃣ B 如果处理不过来 → 释放 CTS
5️⃣ A 立即暂停发送

这就是：

>   **硬件级的“暂停按钮”**

它是电平控制，不是数据协议的一部分。

------

# 四、两种流控方式

## 1️⃣ 硬件流控（RTS/CTS）

✔ 用独立信号线
✔ 响应速度快
✔ 不占用数据带宽
✔ 可靠

需要 5 根线：

-   TX
-   RX
-   GND
-   RTS
-   CTS

------

## 2️⃣ 软件流控（XON / XOFF）

不用额外信号线。

当接收方处理不过来时：

发送一个控制字符：

-   0x13 → 暂停（XOFF）
-   0x11 → 继续（XON）

缺点：

-   占用数据流
-   可能和正常数据冲突
-   实时性不如硬件流控

------

# 五、RS232 为什么有这么多控制线？

因为它最初是为“调制解调器”设计的。

除了 RTS / CTS 还有：

| 信号 | 作用       |
| ---- | ---------- |
| DTR  | 终端准备好 |
| DSR  | 设备准备好 |
| DCD  | 检测到载波 |
| RI   | 电话振铃   |

现代嵌入式基本只用 RTS/CTS。

------

# 六、在 MCU 里的表现

比如：

-   STMicroelectronics STM32
-   Espressif Systems ESP32

UART 外设通常支持：

```c
硬件流控模式：
UART_HWCONTROL_RTS
UART_HWCONTROL_CTS
```

你启用后：

👉 UART硬件自动检测 CTS
👉 自动拉 RTS
👉 不需要软件干预

------

# 七、RS485 为什么没有 RTS/CTS？

因为：

-   RS485 是总线结构
-   通常半双工
-   通过 DE 引脚控制方向
-   协议层自己做仲裁（如 Modbus）

它不是点对点，所以不用这种流控方式。

------

# 八、总结

你记得的那种串口通信：

✅ 是 UART
✅ 通常跑在 RS232 上
✅ 使用 RTS / CTS 硬件流控

------

# 九、你现在需要它吗？

一般情况下：

-   调试串口（115200）→ 不需要
-   高速串口（>1Mbps）→ 可能需要
-   大数据传输 → 建议开启
