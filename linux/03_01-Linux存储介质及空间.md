# Linux存储介质及空间

## 基础

Linux 磁盘管理好坏直接关系到整个系统的性能问题。

文件系统的信息被记录在 ` /etc/fstab ` 文件里。内容如下：

```
<file system> <mount point> <type>  <options>   <dumo>   <psaa>
    设备           挂载点   文件系统类型  选项       频率       顺序
UUID=一串字符         /       ext4   defaults    noatime    0      2
```

使用 ` mount ` 指令可以查看挂载的文件系统，

Linux 磁盘管理常用三个命令为 **df**、**du** 和 **fdisk**。

- **df**（英文全称：disk free）：列出文件系统的整体磁盘使用量
- **du**（英文全称：disk used）：检查磁盘空间使用量
- **fdisk**：用于磁盘分区

### df

|      参数       |                             作用                             |      |
| :-------------: | :----------------------------------------------------------: | ---- |
|       -h        | 以人类可读的方式显示输出结果（例如，使用 KB、MB、GB 等单位） |      |
|       -T        |                      显示文件系统的类型                      |      |
| -t 文件系统类型 |                   只显示指定类型的文件系统                   |      |
|       -i        |                     显示 inode 使用情况                      |      |
|       -H        | 该参数是 `-h` 的变体，但是使用 1000 字节作为基本单位而不是 1024 字节。这意味着它会以 SI（国际单位制）单位（例如 MB、GB）而不是二进制单位（例如 MiB、GiB）来显示磁盘使用情况。 |      |
|       -k        |        这个选项会以 KB 作为单位显示磁盘空间使用情况。        |      |
|       -a        | 该参数将显示所有的文件系统，包括虚拟文件系统，例如 `proc`、`sysfs` 等。如果没有使用该选项，默认情况下，`df` 命令不会显示虚拟文件系统。 |      |
|                 |                                                              |      |

### du

选项与参数：

- -a ：列出所有的文件与目录容量，因为默认仅统计目录底下的文件量而已。
- -h ：以人们较易读的容量格式 (G/M) 显示；
- -s ：仅显示指定目录或文件的总大小，而不显示其子目录的大小。
- -S ：包括子目录下的总计，与 -s 有点差别。
- -k ：以 KBytes 列出容量显示；
- -m ：以 MBytes 列出容量显示

### 磁盘检验fsck（file system check）

用来检查和维护不一致的文件系统。

若系统掉电或磁盘发生问题，可利用fsck命令对文件系统进行检查。

### 磁盘挂载与卸除

Linux 的磁盘挂载使用 `mount` 命令，卸载使用 `umount` 命令。
