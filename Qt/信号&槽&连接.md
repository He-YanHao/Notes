# 信号&槽&连接

Qt 的信号与槽（Signal & Slot）机制是其**核心特性**之一，提供了一种强大、灵活且类型安全的方式来实现**对象之间的通信**。

## 核心概念

**信号 (Signal):**

*   由对象在特定事件发生时**发出**。
*   在 Qt 中，信号是类的成员函数，但**只有声明，没有定义**（由 moc 自动生成）。在类声明中使用 `signals:` 区域声明。
*   信号通常表示“发生了某事”（例如：`clicked()`, `textChanged(QString)`, `valueChanged(int)`）。
*   信号可以被**广播**给所有连接到它的槽。
*   一个信号可以连接到**多个槽**。

**槽 (Slot):**

*   是对信号做出**响应**的普通成员函数。
*   在类声明中使用 `public slots:`、`protected slots:` 或 `private slots:` 区域声明（现代 Qt 也允许将普通成员函数用作槽，但显式声明更清晰）。
*   槽函数具有完整的实现代码，执行信号发出后需要进行的实际操作（例如：`close()`, `setText(QString)`, `updateProgress(int)`）。
*   一个槽可以接收**多个不同信号**发出的信号。

**连接 (Connection):**

*   使用 `QObject::connect()` 函数在**信号**和**槽**之间建立关联。
*   当信号被发出时，所有连接到它的槽函数将按照连接的顺序被**自动调用**。
*   连接是 Qt 实现**松耦合**的关键：发出信号的对象不需要知道哪个对象（如果有的话）会接收它，接收槽的对象也不需要知道是哪个信号触发了它。它们只需要通过 `connect` 建立关系。

## 关键特性和优势

**松耦合 (Loose Coupling):**

*   信号发送者和槽接收者彼此**不了解**对方的细节（具体类型）。发送者只负责发出信号，不知道也不关心谁接收；接收者只负责处理信号，不知道也不关心是谁发出的。
*   这极大地提高了代码的**模块化**和**可维护性**。可以轻松地添加、移除或替换对象，而无需修改通信机制本身。

**类型安全 (Type Safety):**

*   `QObject::connect()` 函数在建立连接时会**检查信号和槽的签名（参数类型和顺序）是否兼容**。
*   如果使用 Qt5 引入的**基于函数指针的新语法**（强烈推荐），编译器就能在**编译期**进行严格的类型检查，防止运行时错误（如参数类型不匹配、信号或槽名拼写错误）。这比旧式的基于字符串的语法（`SIGNAL()`/`SLOT()`宏）安全得多。

**支持多对多通信:**

*   一个信号可以连接**多个槽**。
*   一个槽可以被**多个不同信号**连接。
*   这种灵活性使得事件处理逻辑可以非常清晰地组织和复用。

**线程安全 (Queued Connections):**

*   Qt 支持不同类型的连接方式：
    *   `Qt::AutoConnection` (默认)：如果发送者和接收者在同一线程，行为等同于 `DirectConnection`；否则等同于 `QueuedConnection`。
    *   `Qt::DirectConnection`：信号发出后，槽函数**立即**在发出信号的线程中被调用。
    *   `Qt::QueuedConnection`：信号发出后，一个事件会被放入接收者所在线程的事件队列中。当接收者线程处理该事件时，槽函数才被调用。**这是跨线程通信的安全方式**。
    *   `Qt::BlockingQueuedConnection`：类似 `QueuedConnection`，但信号发送线程会**阻塞**，直到接收者线程执行完槽函数（需谨慎使用，避免死锁）。
    *   `Qt::UniqueConnection`：与 `AutoConnection` 类似，但确保相同信号和槽之间只存在一个连接（防止重复连接）。
*   这种机制使得在多线程程序中安全地传递事件变得非常简单。

**元对象系统 (Meta-Object System):**

*   信号与槽机制依赖于 Qt 的**元对象系统 (Meta-Object System - MOC)**。
*   在编译 Qt 项目时，**元对象编译器 (moc)** 会处理包含 `Q_OBJECT` 宏的头文件。`moc` 会解析 `signals`, `slots`, `Q_PROPERTY` 等关键字，并生成包含这些元信息（如类名、信号签名、槽签名、属性）的额外 C++ 代码（`moc_*.cpp` 文件）。
*   运行时，元对象系统利用这些信息来实现动态特性，如信号发射、槽调用、属性访问、运行时类型信息 (`qobject_cast`) 等。`connect()` 和 `disconnect()` 函数的底层实现也依赖于元对象系统。

## 如何使用 (基本步骤)

**定义信号和槽 (在头文件中):**

```cpp
// sender.h
#include <QObject>
class Sender : public QObject
{
    Q_OBJECT
public:
    explicit Sender(QObject *parent = nullptr);
signals:
    void valueChanged(int newValue); // 信号声明 (无函数体)
};

// receiver.h
#include <QObject>
class Receiver : public QObject
{
    Q_OBJECT
public:
    explicit Receiver(QObject *parent = nullptr);
public slots: // 可以是 private/protected slots
    void handleValueChange(int val); // 槽声明 (有函数体)
};
```

**实现槽函数 (在源文件中):**

```cpp
// receiver.cpp
#include "receiver.h"
#include <QDebug>
Receiver::Receiver(QObject *parent) : QObject(parent) {}
void Receiver::handleValueChange(int val) {
    qDebug() << "Value changed to:" << val;
    // 执行实际的操作...
}
```

**建立连接 (通常在创建对象后):**

```cpp
// main.cpp 或其他地方
#include "sender.h"
#include "receiver.h"
int main(int argc, char *argv[]) {
    QCoreApplication app(argc, argv); // 或 QApplication
    Sender sender;
    Receiver receiver;
    // 强烈推荐使用: Qt5 新语法 (类型安全!)
    QObject::connect(&sender, &Sender::valueChanged,
                     &receiver, &Receiver::handleValueChange);
    // 旧语法 (不推荐，易出错，但有时用于动态连接):
    // QObject::connect(&sender, SIGNAL(valueChanged(int)),
    //                 &receiver, SLOT(handleValueChange(int)));
    // 触发信号 (通常在 Sender 类的某个方法内部)
    sender.valueChanged(42); // 实际中不会直接这样调用，而是由内部逻辑触发
    return app.exec();
}
```

**断开连接 (可选):**

```cpp
// 断开特定信号和特定槽
QObject::disconnect(&sender, &Sender::valueChanged,
                   &receiver, &Receiver::handleValueChange);
// 断开 sender 对象的所有信号与 receiver 对象的所有槽
QObject::disconnect(&sender, nullptr, &receiver, nullptr);
// 断开 sender 对象的所有信号与所有对象的槽
QObject::disconnect(&sender, nullptr, nullptr, nullptr);
```

### 常见应用场景

*   **GUI 事件处理：** 按钮点击 (`clicked()` -> `handleClick()`)，文本输入变化 (`textChanged(QString)` -> `validateInput(QString)`)，滑块移动 (`valueChanged(int)` -> `updateValue(int)`)。
*   **业务逻辑通知：** 数据加载完成 (`dataReady()` -> `processData()`)，网络请求成功/失败 (`finished()` -> `handleResponse()`)，状态改变 (`stateChanged(State)` -> `updateUI(State)`)。
*   **模型/视图通信：** 模型数据改变 (`dataChanged()` -> `updateView()`)。
*   **跨线程通信：** 使用 `Qt::QueuedConnection` 安全地将工作线程的结果或状态更新传递到主线程更新 UI。

### 总结

Qt 的信号与槽机制是其事件驱动编程模型的基石。它通过提供一种**松耦合、类型安全、灵活且支持多线程**的对象间通信方式，极大地简化了应用程序的开发，尤其是事件密集型应用（如 GUI 程序）。理解并熟练运用信号与槽，是掌握 Qt 编程的关键一步。**务必优先使用基于函数指针的新语法 (`&Sender::signal, &Receiver::slot`) 来确保类型安全和编译期检查。**
