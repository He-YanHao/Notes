# 宏在模块化编程中的作用

防止头文件重复包含的宏命名没有官方的强制标准，一般使用最多的是：

```c
【1】#ifdef  FILENAME_H
【2】#define FILENAME_H
		程序段
【3】#endif
```

仅仅是使用 `_` 代替 `.h` 的 `.` ，并将文件字母大写。

作用是：

-   如果没有定义过宏 ` FILENAME_H ` 则说明这是首次编译，执行【2】定义该宏，然后编译程序段。
-   如果已经定义过宏 ` FILENAME_H ` 则说明这不是首次编译，执行【3】直接结束。

从而避免重复定义。



也存在使用 `__FILENAME_H__` 代替 `FILENAME_H` 的用法。

```
【1】#ifdef  __FILENAME_H__
【2】#define __FILENAME_H__
		程序段
【3】#endif
```



最优方式是使用

```c
#pragma once
```

代替，虽然非标准但广泛支持。
